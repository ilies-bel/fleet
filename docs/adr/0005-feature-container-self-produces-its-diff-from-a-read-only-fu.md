# Feature container self-produces its diff from a read-only full git context

Supersedes ADR-0003 and ADR-0004. Each feature is a LINKED git worktree, so its source-only container (mounts <worktree>/<svc> -> /app/<svc>, no .git) cannot resolve 'git diff main...HEAD'. The previously-shipped host-side diff (commit 9182388, git -C worktreePath in the gateway) is WRONG because the gateway is containerized with only the docker socket mounted and cannot see host worktree paths. Decision: give each feature container the WHOLE git context to self-produce its diff by bind-mounting, read-only, (a) the worktree root (so the linked-worktree .git pointer file is present) and (b) the main repo's common git dir / object store (so main and HEAD objects resolve); the gateway then runs 'git -C <worktree-root-mount> --no-optional-locks diff main...HEAD' via docker exec. This differs from the deleted gitlinks override, which mounted the gitdir/commondir but still ran the diff against /app/<subdir> that had no .git, so it never resolved. Trade-off accepted: the full repo object store (all branches/history) is visible read-only inside every feature container; read-only guarantees the shared store cannot be mutated and HEAD/index/locks are never touched, preserving git's same-branch checkout protection. We reject host-FS access for the gateway (re-couples to host paths the container can't see) and reject precomputing a stale diff snapshot (loses live diffs).
