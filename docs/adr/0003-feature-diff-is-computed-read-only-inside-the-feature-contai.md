# Feature diff is computed read-only inside the feature container, not on the host

The dashboard DIFF tab needs git diff main...HEAD for a feature, but the containerized gateway cannot see host worktree paths and feature containers expose source subdirs without their .git. Decision: the gateway computes the diff by exec-ing git READ-ONLY inside the feature container over the existing Docker-socket seam, exposing the worktree's git metadata read-only. We reject (a) giving the gateway host-filesystem access (introduces a new gateway->host seam and re-couples to host paths) and (b) any in-container git writes (would let two writers act on one shared object store, risking branch desync and bypassing git's same-branch checkout protection). Read-only diffs never move HEAD, touch the index, or take the worktree lock, so protection stays intact. Note the topology split: normal repos resolve .git at the repo root above the mounted subdir, while linked worktrees (.worktrees/*) use a .git pointer file referencing the main repo's object store — both must be visible for the read to succeed.
