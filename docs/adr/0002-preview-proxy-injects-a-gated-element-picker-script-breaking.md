# Preview proxy injects a gated element-picker script, breaking byte-transparency

The :3000 preview proxy was documented as transparent (forwards previewed-app HTML verbatim). To support Cursor-style element capture across the cross-origin preview iframe, the proxy now rewrites previewed HTML to inject a picker client script. The script is dormant by default and only activates in capture mode, but it is present in every previewed page. Decision: accept a deliberate, gated break of byte-transparency because there is no other way to read clicks/selectors inside a cross-origin iframe from the dashboard. Trade-off: transparency vs. in-preview authoring UX; we chose authoring UX. Also entails stripping CSP/X-Frame-Options on previewed responses so the iframe + inline picker can run.
