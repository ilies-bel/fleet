# Feature container mounts worktree git metadata read-only so in-container diff resolves

ADR-0003 decided feature diff is read git read-only INSIDE the feature container, but the local compose mounts only the subproject subdir (<worktree>/<dir> -> /app/<dir>), so the worktree's .git is never visible and 'git diff main...HEAD' fails with 'not a git repository'. Decision: on the normal container up path (not just rebuild), bind-mount the worktree root and, for linked worktrees, the referenced per-worktree gitdir + common object store, all read-only, and run 'git -C /app diff main...HEAD'. This satisfies ADR-0003's own 'both must be visible for the read to succeed' precondition by reusing the existing docker-compose.gitlinks.yml override mechanism. We reject moving diff to host-side git because the gateway is containerized (docker run with only the docker socket + ~/.fleet mounted) and cannot see host worktree paths.
