# Migration notes

## 2026-04-16 — Stack-agnostic refactor (epic qa-fleet-pyn)

### Why

The old model treated projects as exactly two services — a `frontend` and a `backend`. That closed enum blocked projects with more than two services, monorepos, and any project where the frontend/backend split did not apply. The refactor replaces the two-slot model with an open `[[services]]` list where each service declares its own stack, port, build command, and run command.

### New layout

```
<project-root>/
├── .fleet/
│   ├── fleet.toml                       (replaces fleet.conf + .fleet-config)
│   ├── fleet.toml.example               (tracked reference copy)
│   ├── shared.env                       (replaces .fleet-shared)
│   ├── shared.env.example               (tracked reference copy)
│   ├── Dockerfile.feature-base.<stack>  (one per stack; generated by fleet init)
│   └── <feature>/
│       ├── info.toml                    (replaces per-feature info file)
│       └── docker-compose.yml
```

`fleet.toml` and all generated files under `.fleet/` are gitignored. Only `*.example` files are tracked.

### CLI surface changes

| Old | New |
|---|---|
| `fleet init <app-root> <branch>` | `fleet init` (no args; reads cwd) |
| `fleet add <name> <branch> [--direct]` | `fleet add <name> [--service svc=path:image ...]` |
| `fleet feature -c <name>` | **REMOVED** |

### Config key mapping

| Old `fleet.conf` key | New `fleet.toml` key |
|---|---|
| `APP_ROOT` | `[project].root` |
| `PROJECT_NAME` | `[project].name` |
| `FRONTEND_DIR` | `[[services]].dir` (service named `frontend`) |
| `BACKEND_DIR` | `[[services]].dir` (service named `backend`) |
| `FRONTEND_PORT` | per-service `port` |
| `BACKEND_PORT` | per-service `port` |
| `BACKEND_BUILD_CMD` | per-service `build` |
| `BACKEND_RUN_CMD` | per-service `run` |
| `PROXY_PORT` | `[ports].proxy` |
| `ADMIN_PORT` | `[ports].admin` |
| `DB_PORT` | `[ports].db` |

### Container and image naming

| Old | New |
|---|---|
| Container `fleet-<feature>` | Container `fleet-<feature>-<service>` (one per service) |
| Image `fleet-feature-base` | Image `fleet-base-<stack>` (one per stack) |

### Upgrade path

No automated migration script is provided. The config format changed too significantly for a mechanical upgrade to be safe.

If you have an existing project managed by the previous version:

```bash
# 1. Tear down all existing containers and remove old config
fleet rm --nuke

# 2. Remove old flat config files from your project repo
rm fleet.conf .fleet-config .fleet-shared 2>/dev/null || true

# 3. Re-initialise from scratch
cd <project-root>
fleet init
```

`fleet init` will scan your service directories, detect stacks, write `.fleet/fleet.toml`, and regenerate all Dockerfiles.

# Migration: qa-fleet → fleet (v0.2)

v0.2 renames all `qa-*` identifiers to `fleet-*` for consistency with the CLI and npm package name. This is a breaking change with no automatic migration path.

## Before upgrading

On the old version (still installed), tear down everything:

```bash
fleet rm --nuke
```

This removes all `qa-<feature>` containers, `qa-gateway`, `qa-net` network, and `.qa-config`. You may also want to manually delete `qa-fleet.conf` and `.qa-worktrees/` in your app repo.

## After upgrading

```bash
npm i -g @ilies-bel/fleet
fleet init <app-root> <branch>
```

The wizard writes `fleet.conf` (not `qa-fleet.conf`). State lives in `.fleet-config`, worktrees in `.fleet-worktrees/`. Containers are named `fleet-<name>` on network `fleet-net`. Admin API moved from `/_qa/` to `/_fleet/`.

## Rename reference

| Old | New |
|---|---|
| `qa-fleet.conf` | `fleet.conf` |
| `.qa-config` | `.fleet-config` |
| `.qa-worktrees/` | `.fleet-worktrees/` |
| `.qa-shared` | `.fleet-shared` |
| `qa-net` | `fleet-net` |
| `qa-gateway` | `fleet-gateway` |
| `qa-<name>` | `fleet-<name>` |
| `/_qa/` | `/_fleet/` |
| `__QA_BACKEND_URL__` | `__FLEET_BACKEND_URL__` |
| `__QA_APP_URL__` | `__FLEET_APP_URL__` |
| `X-QA-Feature` header | `X-Fleet-Feature` header |

OAuth callback URL is unchanged: `http://localhost:4000/auth/callback`.
